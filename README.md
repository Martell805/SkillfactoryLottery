# Краткое описание функционала

Система позволяет проводить лотереи, в которых будут учавствавать пользователи.
Лотереи могут проводиться вручную или быть запланированы. 
Лотереи могут проводиться как моментально, так и с вытаскиванием каждого бочонка.
Выигрыш для каждого билета автоматически рассчитывается системой.

Пользователям предоставляется интерфейс, позволяющий покупать любые билеты и получать информацию о тиражах и своих билетах и операциях.
Администраторам предоставляется возможность полного контроля всех сущностей через CRUD API, а также выгрузка CSV//JSON отчётов.

# Задачи:

### Лотерея:
- Создание тиража (эндпоинт администратора)
- Создание ежедневного (вынести cron в конфигурацию приложения) тиража (в программе, через scheduler)
- Проведение лотереи: эндпоинт для вытягивания следующего бочонка и эндпоинт для вытягивания всех оставшихся бочонков
- Проведение ежедневной (вынести cron в конфигурацию приложения) лотереи (в программе, через scheduler)

### Пользователь:
- Получение всех тиражей с фильтрами по статусу, формату и времени проведения
- Получение всех не купленных билетов по тиражу (если билеты не создаются пользователем)
- Покупка подготовленного заранее билета по id
- Покупка билета с созданием в рамках тиража
- Получить весь результат тиража
- Получить последний бочонок тиража
- Получить историю своих операций (за определённое время)
- Получить свои билеты (за определённое время и за определённый тираж)

### Админ:
- Управление тиражами (CRUD)
- Управление результатами тиражей (CRUD)
- Управление пользователями (CRUD)
- Управление билетами (CRUD)
- Управление операциями (CRUD)

### Отчёты (тоже админ):
- Выгрузить CSV//JSON (2 эндпоинта) отчёт по операциям с фильтрами по пользователям, тиражам и времени покупки
- Выгрузить CSV//JSON (2 эндпоинта) отчёт по билетам с фильтрами по пользователям, тиражам и времени покупки и проведения
- Выгрузить CSV//JSON (2 эндпоинта) отчёт по тиражам (и их результатам) с фильтрами по времени проведения, статусу и формату
